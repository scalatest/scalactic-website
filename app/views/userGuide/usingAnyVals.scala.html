Using AnyVals

Scalactic provides a rich suite of AnyVal types in the org.scalactic.anyvals package. These types enforce value constraints via a macro that checks literals at compile time, providing compile-time safety with zero runtime overhead. AnyVals are value classes (they extend AnyVal), meaning they erase to their underlying primitive at runtime.

AnyVals are the original constrained-value mechanism in Scalactic. They work with Scala 2 and Scala 3, and are compatible with ScalaTest versions before 3.3.0 (which introduced opaque types as an alternative). If you're using Scala 3.3.0+, consider the newer opaque types for stronger type-level separation while keeping an erased runtime representation.

Why AnyVals?

AnyVals provide several benefits:

  • Compile-time safety for literals: Invalid literal values are caught at compile time via a macro, producing a compiler error.
  • Zero runtime overhead: Being value classes, AnyVals erase to their underlying types at runtime.
  • Self-documenting code: A method signature using PosInt clearly communicates that only positive integers are expected.
  • Broad compatibility: Works with Scala 2.x, Scala 3.x, and ScalaTest versions before 3.3.0.

Numeric AnyVals

Scalactic provides a comprehensive set of numeric AnyVals across Int, Long, Float, and Double. Each type enforces a specific constraint:

ConstraintTypes
Positive (greater than zero) PosInt, PosLong, PosFloat, PosDouble
Non-negative (zero or greater) PosZInt, PosZLong, PosZFloat, PosZDouble
Negative (less than zero) NegInt, NegLong, NegFloat, NegDouble
Non-positive (zero or less) NegZInt, NegZLong, NegZFloat, NegZDouble
Non-zero NonZeroInt, NonZeroLong, NonZeroFloat, NonZeroDouble
Finite (not NaN or infinity) FiniteFloat, FiniteDouble
Finite and positive PosFiniteFloat, PosFiniteDouble
Finite and non-negative PosZFiniteFloat, PosZFiniteDouble
Finite and negative NegFiniteFloat, NegFiniteDouble
Finite and non-positive NegZFiniteFloat, NegZFiniteDouble
Finite and non-zero NonZeroFiniteFloat, NonZeroFiniteDouble

Constructing numeric AnyVals with literals

Each numeric AnyVal provides an apply method that is implemented as a macro. When you pass a literal value, the macro checks the value at compile time. If the value satisfies the constraint, compilation succeeds. If not, a compiler error is raised:

import org.scalactic.anyvals.PosInt


// Valid: compiles fine
val positive: PosInt = PosInt(42)


// Compiler error: PosInt.apply can only be invoked on a positive (n > 0) integer literal, like PosInt(42).
val bad: PosInt = PosInt(-5)

Similarly, for non-negative values:

import org.scalactic.anyvals.PosZInt


val nonNeg: PosZInt = PosZInt(0)    // OK: zero is non-negative

val alsoNonNeg: PosZInt = PosZInt(10) // OK


// Compiler error: PosZInt.apply can only be invoked on a non-negative (n >= 0) integer literal, like PosZInt(42).
val bad: PosZInt = PosZInt(-1)

Handling variables with from

The macro can only check literal values — it cannot determine the validity of variables at compile time. When the value comes from a variable (e.g., from a function argument, user input, or computation), use the from factory method instead, which returns an Option[T] and validates at runtime:

import org.scalactic.anyvals.PosInt


def process(count: Int): Unit = {

  PosInt.from(count) match {

    case Some(pi) => // count was positive — use pi.value

    case None     => // count was not positive — handle the error

  }

}


// You can also use the Option directly:
val maybePos: Option[PosInt] = PosInt.from(someVariable)

This gives you compile-time safety for literals (via the macro) and runtime safety for variables (via from).

Implicit widening conversions

AnyVals provide implicit widening conversions, so you can pass them where the underlying primitive type is expected:

def add(a: Int, b: Int): Int = a + b


add(PosInt(1), PosInt(2)) // Implicit conversion from PosInt to Int

Character and String AnyVals

Scalactic provides constrained character and string types:

  • NumericChar — a Char that is a numeric digit ('0' through '9')
  • NumericString — a String consisting entirely of digits
import org.scalactic.anyvals.NumericChar

import org.scalactic.anyvals.NumericString


val digit: NumericChar = NumericChar('5') // OK

val number: NumericString = NumericString("12345") // OK


// Compiler errors for invalid literals
val badChar: NumericChar = NumericChar('a')
val badString: NumericString = NumericString("abc")

Collection AnyVals

Non-empty collection types ensure collections always contain at least one element:

  • NonEmptyString — a String that is not empty
  • NonEmptySeq[A] — a Seq with at least one element
  • NonEmptyList[A] — a List with at least one element
  • NonEmptyVector[A] — a Vector with at least one element
  • NonEmptySet[A] — a Set with at least one element
  • NonEmptyMap[K, V] — a Map with at least one entry
  • NonEmptyArray[T] — an Array with at least one element

Example with NonEmptyString

import org.scalactic.anyvals.NonEmptyString


val name: NonEmptyString = NonEmptyString("Alice") // OK


// Compiler error: NonEmptyString.apply can only be invoked on a non-empty string literal
// like NonEmptyString("hello").
val bad: NonEmptyString = NonEmptyString("")

Building NonEmptyList with End

The NonEmptyList AnyVal provides a convenient :: operator for construction, terminated by End:

import org.scalactic.anyvals.NonEmptyList

import org.scalactic.anyvals.End


val numbers = 1 :: 2 :: 3 :: End

CompileTimeAssertions Trait

CompileTimeAssertions is primarily for library authors who implement macro-based constrained AnyVals. It provides helper methods such as ensureValidIntLiteral, ensureValidLongLiteral, and related methods for other literal-supported types. These helpers are used inside macro implementations to produce compiler errors for invalid or non-literal arguments.

For day-to-day usage of built-in AnyVals (like PosInt and NonEmptyString), you do not need this trait.

import org.scalactic.anyvals.CompileTimeAssertions

import reflect.macros.Context


object OddIntMacro extends CompileTimeAssertions {

  def isValid(i: Int): Boolean = i.abs % 2 == 1



  def apply(c: Context)(value: c.Expr[Int]): c.Expr[OddInt] = {

    val notValidMsg = "OddInt.apply can only be invoked on odd Int literals, like OddInt(3)."

    val notLiteralMsg = "OddInt.apply can only be invoked on Int literals. Please use OddInt.from instead."



    ensureValidIntLiteral(c)(value, notValidMsg, notLiteralMsg)(isValid)

    c.universe.reify { OddInt.ensuringValid(value.splice) }

  }

}

AnyVals vs Opaque Types

Both AnyVals and opaque types provide constrained values, but they differ in important ways:

FeatureAnyValsOpaque Types
Literal validation Compile time (via macro) Compile time for literals
Variable validation Runtime via from (returns Option) Runtime at construction boundary
Error type Compiler error (literals) / Option (variables) Compiler error (literals) / runtime validation failure for non-literals
Scala version support Scala 2.x, Scala 3.x Scala 3.x only
ScalaTest compatibility All versions 3.3.0+
Runtime overhead None (value class erasure) None for representation (erased to underlying type)

For more on opaque types, see the Using Opaque Types guide.

Where to find the API

All AnyVals live in the org.scalactic.anyvals package. The list below is a representative subset (not exhaustive):

  • PosInt — positive Ints
  • PosZInt — non-negative Ints
  • NegInt — negative Ints
  • NegZInt — non-positive Ints
  • NonZeroInt — non-zero Ints
  • FiniteFloat — finite Floats (not NaN or infinity)
  • FiniteDouble — finite Doubles (not NaN or infinity)
  • PosFiniteFloat — positive finite Floats
  • PosFiniteDouble — positive finite Doubles
  • PosZFiniteFloat — non-negative finite Floats
  • PosZFiniteDouble — non-negative finite Doubles
  • NegFiniteFloat — negative finite Floats
  • NegFiniteDouble — negative finite Doubles
  • NegZFiniteFloat — non-positive finite Floats
  • NegZFiniteDouble — non-positive finite Doubles
  • NonZeroFiniteFloat — non-zero finite Floats
  • NonZeroFiniteDouble — non-zero finite Doubles
  • NumericChar — numeric characters
  • NumericString — numeric strings
  • NonEmptyString — non-empty strings
  • NonEmptyList — non-empty lists
  • NonEmptyVector — non-empty vectors
  • NonEmptySet — non-empty sets
  • NonEmptyMap — non-empty maps
  • NonEmptyArray — non-empty arrays
  • End — endpoint for NonEmptyList construction
  • CompileTimeAssertions — macro helper trait for implementing custom AnyVals with compile-time literal validation
