Using Opaque Types

Scalactic provides a suite of opaque types for Scala 3, starting with Scalactic 3.3.0. These types model constrained values (such as positive numbers and non-empty collections) while preserving an erased runtime representation.

In traditional Scala, you might represent a positive integer as Int, but there's nothing stopping someone from passing -5 where a positive value is expected. With opaque types, you can use PosInt to represent that constraint explicitly in method signatures.

Why Opaque Types?

Opaque types provide several benefits:

  • Compile-time checking for literals: Literal arguments to apply are validated at compile time.
  • Self-documenting code: A method signature using PosInt clearly communicates that only positive integers are accepted.
  • Runtime validation helpers for non-literals: Methods like from, ensuringValid, and tryingValid handle values known only at runtime.

Numeric Opaque Types

Scalactic provides a comprehensive set of numeric opaque types across Int, Long, Float, and Double. Each type enforces a specific constraint:

ConstraintTypes
Positive (greater than zero) PosInt, PosLong, PosFloat, PosDouble
Non-negative (zero or greater) PosZInt, PosZLong, PosZFloat, PosZDouble
Negative (less than zero) NegInt, NegLong, NegFloat, NegDouble
Non-positive (zero or less) NegZInt, NegZLong, NegZFloat, NegZDouble
Non-zero NonZeroInt, NonZeroLong, NonZeroFloat, NonZeroDouble
Finite (not NaN or infinity) FiniteFloat, FiniteDouble
Finite and positive PosFiniteFloat, PosFiniteDouble
Finite and non-negative PosZFiniteFloat, PosZFiniteDouble
Finite and negative NegFiniteFloat, NegFiniteDouble
Finite and non-positive NegZFiniteFloat, NegZFiniteDouble
Finite and non-zero NonZeroFiniteFloat, NonZeroFiniteDouble

Constructing opaque types

For numeric opaque types, apply validates literal arguments at compile time. Invalid literals (or non-literals passed to literal-only overloads) produce compiler errors:

import org.scalactic.opaquetypes.PosInts.PosInt


// Valid construction
val positive: PosInt = PosInt(42)


// This fails with compiler error
val bad: PosInt = PosInt(-5)

Similarly, for non-negative values:

import org.scalactic.opaquetypes.PosInts.PosZInt


val nonNeg: PosZInt = PosZInt(0)    // OK: zero is non-negative

val alsoNonNeg: PosZInt = PosZInt(10) // OK


// This fails with compiler error
val bad: PosZInt = PosZInt(-1)

Handling non-literal values

When the value is not known at compile time (for example, function parameters or user input), use runtime factory methods:

import org.scalactic.opaquetypes.PosInts.PosInt

import scala.util.Try


def parseCount(n: Int): Unit = {

  // Option-based runtime validation

  val maybe: Option[PosInt] = PosInt.from(n)



  // Throws AssertionError if invalid

  // val strict: PosInt = PosInt.ensuringValid(n)



  // Try-based runtime validation

  val attempted: Try[PosInt] = PosInt.tryingValid(n)

}

Character Opaque Types

Scalactic also provides:

  • NumericChar — a Char constrained to digits ('0' through '9')
  • NumericString — a String constrained to numeric digits
import org.scalactic.opaquetypes.Numerics.NumericChar

import org.scalactic.opaquetypes.Numerics.NumericString


val digit: NumericChar = NumericChar('5') // OK

val digits: NumericString = NumericString("12345") // OK


// This fails with compiler error
val bad: NumericChar = NumericChar('a')
val badString: NumericString = NumericString("12a")

Collection Opaque Types

Non-empty collection types ensure collections always contain at least one element:

  • NonEmptyString — a String that is not empty
  • NonEmptySeq[A] — a Seq with at least one element
  • NonEmptyList[A] — a List with at least one element
  • NonEmptyVector[A] — a Vector with at least one element
  • NonEmptySet[A] — a Set with at least one element
  • NonEmptyMap[K, V] — a Map with at least one entry
  • NonEmptyArray[T] — an Array with at least one element

Example with NonEmptyString

import org.scalactic.opaquetypes.NonEmpties.NonEmptyString


val name: NonEmptyString = NonEmptyString("Alice") // OK


// This fails with compiler error
val bad: NonEmptyString = NonEmptyString("")

How Using Opaque Types Help in Tests

A common benefit of opaque types is that they can eliminate an entire class of unit tests: defensive checks for impossible domain values in internal methods.

Without PosInt, a method that accepts Int usually needs a runtime guard, and you typically add tests for negative/zero input:

class AccountService {

  def addBonus(points: Int): Int = {

    require(points > 0, "points must be positive")

    points + 10

  }

}



// Typical tests with Int API:
// - accepts positive input
// - throws on zero
// - throws on negative input

With PosInt, that defensive check is moved to construction boundaries, so the method itself no longer needs negative-input tests:

import org.scalactic.opaquetypes.PosInts.PosInt


class AccountService {

  def addBonus(points: PosInt): Int =

    points.value + 10

}



// Positive-path test:
// addBonus(PosInt(5)) == 15


// Invalid literal does not compile:
// addBonus(PosInt(-1))

In practice, this means you still test boundary adapters (for example, parsing request data with PosInt.from), but you can skip repetitive "negative Int" tests for internal methods that already require PosInt.

Where to find the API

All opaque types live in the org.scalactic.opaquetypes package. The list below is a representative subset (not exhaustive):

  • PosInts — positive and non-negative Int-based opaque types
  • PosLongs — positive and non-negative Long-based opaque types
  • PosFloats — positive and finite-related Float-based opaque types
  • PosDoubles — positive and finite-related Double-based opaque types
  • NegInts — negative and non-positive Int-based opaque types
  • NegLongs — negative and non-positive Long-based opaque types
  • NegFloats — negative and finite-related Float-based opaque types
  • NegDoubles — negative and finite-related Double-based opaque types
  • NonZeroInts — non-zero Int-based opaque types
  • NonZeroLongs — non-zero Long-based opaque types
  • NonZeroFloats — non-zero and finite-related Float-based opaque types
  • NonZeroDoubles — non-zero and finite-related Double-based opaque types
  • Finites — finite Float and Double-based opaque types
  • Numerics — numeric character and string opaque types
  • NonEmpties — non-empty collection and string opaque types
